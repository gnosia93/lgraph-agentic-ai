100B급 대형 멀티모달 모델(VLM)을 교사(Teacher)로 삼아 1B급 소형 정제 모델(Student)을 만드는 과정은 AI 인프라와 데이터 엔지니어링의 정수가 필요한 '지식 증류(Knowledge Distillation)' 및 '합성 데이터 생성(Synthetic Data Generation)' 프로젝트입니다.
특히 멀티모달(텍스트+이미지 등) 영역은 단순 텍스트보다 정렬(Alignment)이 까다롭기 때문에 전략적인 접근이 필요합니다. 

### 1단계: 교사 모델(100B VLM)을 통한 데이터 정제 및 합성 ###
1B 모델은 파라미터가 작기 때문에 지저분한 데이터를 학습하면 성능이 급격히 망가집니다. 100B 모델을 일종의 **'데이터 공장 및 정제소'**로 활용해야 합니다.

* 고품질 캡셔닝 및 QA 세트 생성 (시각적 지식 증류)
  *	대규모 이미지 데이터셋(예: LAION 등)을 준비한 뒤, 100B VLM에게 각 이미지에 대한 극도로 상세한 설명(Dense Captioning)과 복잡한 추론 질문/답변 쌍(VQA)을 생성하게 합니다.
  *	프롬프트 엔지니어링: "생각의 흐름(CoT, Chain-of-Thought)을 포함하여 이 이미지의 인과관계를 단계별로 설명해줘" 형태로 100B의 추론 프로세스 자체를 데이터에 녹여냅니다.
* 데이터 필터링 (품질 검증)
  * 100B 모델이 뱉은 합성 데이터 중 리워드 모델(Reward Model)이나 독성/환각 스캔을 거쳐 품질 점수가 높은 상위 데이터만 정제하여 1B 모델의 학습셋으로 구축합니다.


### 2단계: 소형 모델(1B) 아키텍처 및 정렬(Alignment) 설계 ###
1B 모델은 이미지와 텍스트를 모두 이해할 수 있는 경량 아키텍처로 세팅해야 합니다. 최근 트렌드는 LLaVA 스타일이나 SigLIP + 경량 LLM 컴포넌트 조합입니다.
* 비전 인코더 활용: 이미지 특징 추출은 처음부터 학습시키지 않고, 이미 검증된 오픈소스 비전 인코더(예: SigLIP-SO400M 또는 CLIP-ViT-L)를 고정(Freeze)하여 사용합니다.
* 프로젝션 레이어(연결부) 훈련: 비전 인코더가 뽑은 이미지 벡터를 1B 언어 모델이 이해할 수 있는 차원(Dimension)으로 변환해 주는 가벼운 MLP(Multi-Layer Perceptron) 레이어를 설계합니다.
* 언어 모델(Backbone): 대형 모델에서 텍스트 지식이 잘 전수될 수 있도록 Llama-3-1B나 Qwen2.5-1.5B 같은 검증된 경량 백본을 기반으로 시작합니다.

### 3단계: 인프라 기반 분산 학습 전략 ###
1B 모델 학습이지만 멀티모달 데이터(고해상도 이미지 텐서)는 메모리를 엄청나게 잡아먹기 때문에 다음과 같이 학습을 분기해야 합니다.

#### ① 1단계: 프로젝션 정렬 학습 (Freeze LLM) ####
*	목표: 이미지와 텍스트 벡터 공간 맞추기
*	인프라: 1B 모델 자체는 작으므로 PyTorch Native FSDP를 사용해 싱글 노드 멀티 GPU 환경(training-on-eks 내 L40S 혹은 A100 클러스터)에서 빠르게 끝낼 수 있습니다. 이 단계에서는 LLM 가중치는 잠그고 연결 부위만 학습시킵니다.

#### ② 2단계: 전체 파인튜닝 및 완화된 증류 (Full Fine-tuning) ####
* 목표: 100B가 만든 고품질 멀티모달 QA 데이터를 기반으로 1B 모델 전체를 동시 학습.
* 핵심 기술 (Logit Distillation): 100B 모델이 뱉는 최종 답변의 텍스트뿐만 아니라, 각 토큰의 확률 분포(Logits)를 1B 모델이 모방하도록 Loss Function(KL-Divergence)을 설계하면 1B 모델의 성능이 단순 파인튜닝보다 훨씬 가파르게 상승합니다.
* 인프라 팁: 이미지 텐서와 대규모 배치 사이즈 처리를 위해 DeepSpeed ZeRO-Stage 2/3와 EFA 네트워크를 활성화하여 노드 간 통신 병목을 제거해야 합니다.

### 프로젝트 성공을 위한 퀵 체크리스트 ###
* 100B 추론 인프라 확보: 데이터 합성을 위해 100B 모델을 장기간 대량으로 찔러야 하므로, Bedrock의 Batch Inference(비동기 배치 처리, 비용 50% 절감)를 쓰거나 infer-on-eks 인프라에서 FP8로 경량화하여 vLLM 스루풋을 극한으로 끌어올려 데이터를 먼저 뽑는다.
* 토큰 분석기(Tokenizer) 일치: 가능하다면 교사 모델과 학생 모델의 토크나이저를 일치시키거나 호환 레이어를 두어야 로짓(Logit) 증류 시 수학적 매핑이 꼬이지 않습니다.
* 평가 체계: 완성된 1B 모델의 성능은 MM-Vet이나 MMBench 같은 멀티모달 벤치마크셋을 활용해 교사 모델 대비 몇 %의 성능(예: Teacher 대비 75% 달성 목표)을 내는지 정량적으로 트래킹해야 합니다.

```
1. 데이터 합성 페이즈: '비용'과 '속도'의 타협점 찾기
100B 모델로 수십만 장의 이미지에 대한 캡션과 VQA 데이터를 생성하려면 인프라 비용과 시간이 어마어마하게 듭니다.
•	전략 1: AWS Bedrock의 Batch Inference 기능을 활용하면, 실시간 API 대비 가격을 50% 할인받으면서 비동기(최대 24시간 이내 완료)로 대량의 학습 데이터를 뽑아낼 수 있습니다.
•	전략 2: 만약 자체 GPU 클러스터가 있다면, 100B 모델을 **FP8(8비트 부동소수점)**로 양자화하여 vLLM 기반 멀티노드 파이프라인(infer-on-eks)에 올린 뒤 가동률을 100%로 쥐어짜서 단기간에 데이터를 긁어모아야 합니다.

2. 정렬(Alignment) 페이즈: 왜 LLaVA 구조인가?
1B 소형 모델이 멀티모달을 이해하게 만드는 가장 효율적인 아키텍처는 ‘커넥터(Connector)’ 구조입니다.
•	처음부터 이미지와 텍스트를 동시에 이해하는 거대한 모델을 만드는 것이 아니라, 이미 세상에 나와 있는 최고의 눈(Vision Encoder: CLIP이나 SigLIP)과 최고의 입(LLM: Llama-3-1B 등)을 준비합니다.
•	그리고 그 사이를 연결하는 아주 작은 선(Linear Layer 또는 MLP Projector)만 먼저 학습시키는 것입니다. 이 방식(LLaVA 스타일)을 쓰면 1B 모델이라도 단 몇 장의 GPU로 며칠 만에 멀티모달 모델로 진화시킬 수 있습니다.

3. 학습(Training) 페이즈: 텍스트 너머의 '로짓(Logit) 증류'
100B 교사 모델이 뱉은 정답 텍스트만 보고 1B 모델을 학습시키면, 1B는 교사 모델의 '겉핥기'만 배우게 됩니다.
•	진짜 고수들은 100B 모델이 다음 단어를 예측할 때 계산한 토큰별 확률 분포(Logits) 자체를 데이터셋에 같이 저장합니다.
•	1B 모델을 학습시킬 때 "이 정답을 맞춰라"뿐만 아니라, **"100B 교사가 고민했던 각 단어별 확률의 뉘앙스(Soft Label)까지 똑같이 닮아라"**고 손실 함수(Loss)를 주면, 정제 모델의 추론 능력이 비약적으로 상승합니다.
```
