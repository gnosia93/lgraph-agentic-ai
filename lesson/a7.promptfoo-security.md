## promptfoo로 모델 취약점 점검하기 ##
LLM 취약점 진단 및 보안 테스트 역시 promptfoo 의 redteam 기능을 활용하면 쉽게 테스트 할 수 있습니다. promptfoo는 개발자가 직접 공격 시나리오를 짜지 않아도, AI(Adversarial Generator)가 알아서 내 프롬프트와 LLM을 해킹(레드팀 공격)해 보며 취약점을 찾아내는 자동화 기능을 제공합니다.

## 🛠️ Promptfoo 보안 테스트 3단계 ##
### 1단계: 레드팀 테스트 환경 초기화 ###
터미널을 열고 내 프로젝트 폴더에서 아래 명령어를 입력합니다. 보안 테스트를 위한 전용 설정 파일(promptfoo.yaml)을 만들어주는 마법사(Wizard)가 실행됩니다.
```
npx promptfoo@latest redteam init
```
•	주의: 실행 과정에서 테스트할 내 LLM의 API Key와 타겟 모델(예: GPT-4o, Claude 3.5 Sonnet 등)을 지정해야 합니다.

### 2단계: 원하는 보안 취약점(Plugin) 선택하기 ###
init 명령어를 실행하면 화면에 "어떤 보안 위협을 테스트할래?"라고 물어보며 체크박스가 뜹니다. Promptfoo가 제공하는 대표적인 보안 플러그인은 다음과 같습니다.

| 보안 테스트 항목 (Plugins) | 설명 |
| :--- | :--- |
| **`jailbreak`** | 시스템 프롬프트를 탈옥하여 악의적인 답변을 유도하는지 테스트 |
| **`pii`** | 이메일, 주민번호 등 개인정보를 유출하는지 테스트 |
| **`sql-injection / cve`** | 악성 코드나 시스템 명령어를 주입해 보안을 뚫을 수 있는지 테스트 |
| **`harmful`** | 혐오 발언, 사기, 범죄 방법 등 유해한 콘텐츠를 생성하는지 테스트 |
| **`indirect-prompt-injection`** | 외부 입력 데이터를 통해 간접적으로 해킹당하는지 테스트 |

•	마법사에서 원하는 항목을 체크하면 promptfoo.yaml 파일의 plugins 항목에 자동으로 등록됩니다.

### 3단계: 보안 테스트(공격) 시작하기 ###
설정이 끝났다면 아래 명령어로 AI에게 내 시스템을 공격하라고 명령합니다.
```
npx promptfoo@latest redteam run
```
•	원리: Promptfoo 내부의 전용 공격용 AI(Adversarial Generator)가 수십~수백 개의 정교한 우회 프롬프트를 생성해 내 LLM에 무차별 대입(Fuzzing)을 시작합니다.
•	내 LLM이 낚여서 보안 규정을 위반하는 답변을 출력하면 '실패(Fail)'로 기록됩니다.

### 📊 4단계: 보안 보고서(결과) 확인하기 ###
테스트가 끝나면 터미널에 스코어가 뜨고, 아래 명령어로 웹 브라우저에서 시각화된 대시보드를 볼 수 있습니다.
```
npx promptfoo@latest view
```
•	어떤 프롬프트 공격에 내 시스템이 뚫렸는지, 어떤 취약점(탈옥 등)에 취약한지 한눈에 보여주는 보안 성적표가 제공됩니다. 개발자는 이 취약한 답변들을 보고 시스템 프롬프트(System Prompt)에 "절대 ~한 답변은 하지 말 것"이라는 방어벽을 추가하면 됩니다.

💡 한 줄 요약
**redteam init**으로 내 모델과 원하는 취약점을 고르고, **redteam run**으로 모의 해킹을 보낸 뒤, **view**로 보안 뚫린 곳을 고치면 끝입니다!

## AWS Bedrock 모델 테스트 ##

### 1. 환경 변수(AWS 인증) 설정하기 ###
Promptfoo가 내 AWS 계정의 Bedrock에 접근할 수 있도록 터미널에 AWS 자격 증명과 리전을 입력해 줍니다.
```
export AWS_ACCESS_KEY_ID="내_AWS_액세스_키"
export AWS_SECRET_ACCESS_KEY="내_AWS_시크릿_키"
export AWS_REGION="us-east-1" # 예: 버지니아 북부
```

### 2. promptfoo.yaml 보안 설정 파일 작성 ###
프로젝트 폴더에 promptfoo.yaml 파일을 만들고 아래와 같이 작성합니다. 
```
# promptfoo.yaml
redteam:
  # 1. 테스트 대상 (AWS Bedrock Claude 3.5 Haiku)
  providers:
    - id: bedrock:anthropic.claude-3-5-haiku-20241022-v1:0
      config:
        region: us-east-1
        # 만약 시스템 프롬프트(페르소나)가 있다면 여기에 방어벽을 적어줍니다.
        systemPrompt: "너는 친절한 고객 상담 AI야. 회사 내부 정보나 개인정보는 절대 발설하면 안 돼."

  # 2. 어떤 보안 공격을 시도할 것인가?
  plugins:
    - jailbreak                  # 탈옥 및 우회 시도
    - pii:direct                 # 개인정보 유출 테스트
    - harmful:cyberattacks       # 사이버 공격 가이드 요구 테스트
    - harmful:insults            # 욕설 및 비하 발언 유도 테스트
    - contract                   # 허위 계약이나 사기 유도 테스트

  # 3. 공격용 프롬프트를 몇 개나 만들어서 쏠 것인가? (플러그인당 5개씩 생성)
  numTests: 5
```

### 3. 보안 테스트 실행 및 결과 확인 ###
설정 파일 저장이 끝났다면 터미널에 아래 명령어를 순서대로 입력합니다.

#### ① 모의 해킹(레드팀) 실행 ####
```
npx promptfoo@latest redteam run
```
•	동작 원리: Promptfoo가 자동으로 설정된 플러그인에 맞는 '공격용 프롬프트'들을 무차별 생성한 뒤, AWS Bedrock에 있는 Claude 3.5 Haiku 모델로 전송합니다.
•	하이쿠 모델의 답변을 분석하여 보안 규정을 위반(Fail)했는지 안전하게 방어(Pass)했는지 채점합니다.

#### ② 결과 대시보드 확인 ####
```
npx promptfoo@latest view
```
•	테스트가 끝나면 로컬 웹 서버가 열리면서 어떤 공격 프롬프트에 Haiku 모델이 뚫렸는지(Jailbroken) 시각적으로 예쁘게 보여줍니다.

Claude 3.5 Haiku는 속도가 매우 빠르고 비용이 저렴해서 대규모 보안  fuzzing 테스트(numTests를 20~30 이상으로 올리는 것)를 부담 없이 돌리기에 가장 적합한 모델입니다.
