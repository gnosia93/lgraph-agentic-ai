LLM 취약점 진단 및 보안 테스트 도구인 Promptfoo에서 보안 테스트(Security Test)를 수행하는 핵심은 promptfoo redteam 기능을 사용하는 것입니다.
Promptfoo는 개발자가 직접 공격 시나리오를 짜지 않아도, AI가 알아서 내 프롬프트와 LLM을 해킹(레드팀 공격)해 보며 취약점을 찾아내는 자동화 툴입니다.
가장 빠르고 표준적인 보안 테스트 방법을 단계별로 쉽게 정리해 드릴게요.
🛠️ Promptfoo 보안 테스트 3단계
1단계: 레드팀 테스트 환경 초기화
터미널을 열고 내 프로젝트 폴더에서 아래 명령어를 입력합니다. 보안 테스트를 위한 전용 설정 파일(promptfoo.yaml)을 만들어주는 마법사(Wizard)가 실행됩니다.
npx promptfoo@latest redteam init

•	주의: 실행 과정에서 테스트할 내 LLM의 API Key와 타겟 모델(예: GPT-4o, Claude 3.5 Sonnet 등)을 지정해야 합니다.
2단계: 원하는 보안 취약점(Plugin) 선택하기
init 명령어를 실행하면 화면에 "어떤 보안 위협을 테스트할래?"라고 물어보며 체크박스가 뜹니다. Promptfoo가 제공하는 대표적인 보안 플러그인은 다음과 같습니다.
보안 테스트 항목 (Plugins)	설명
jailbreak	시스템 프롬프트를 탈옥하여 악의적인 답변을 유도하는지 테스트
pii	이메일, 주민번호 등 개인정보를 유출하는지 테스트
sql-injection / cve	악성 코드나 시스템 명령어를 주입해 보안을 뚫을 수 있는지 테스트
harmful	혐오 발언, 사기, 범죄 방법 등 유해한 콘텐츠를 생성하는지 테스트
indirect-prompt-injection	외부 입력 데이터를 통해 간접적으로 해킹당하는지 테스트
•	마법사에서 원하는 항목을 체크하면 promptfoo.yaml 파일의 plugins 항목에 자동으로 등록됩니다.
3단계: 보안 테스트(공격) 시작하기
설정이 끝났다면 아래 명령어로 AI에게 내 시스템을 공격하라고 명령합니다.
npx promptfoo@latest redteam run

•	원리: Promptfoo 내부의 전용 공격용 AI(Adversarial Generator)가 수십~수백 개의 정교한 우회 프롬프트를 생성해 내 LLM에 무차별 대입(Fuzzing)을 시작합니다.
•	내 LLM이 낚여서 보안 규정을 위반하는 답변을 출력하면 '실패(Fail)'로 기록됩니다.
📊 4단계: 보안 보고서(결과) 확인하기
테스트가 끝나면 터미널에 스코어가 뜨고, 아래 명령어로 웹 브라우저에서 시각화된 대시보드를 볼 수 있습니다.
npx promptfoo@latest view

•	어떤 프롬프트 공격에 내 시스템이 뚫렸는지, 어떤 취약점(탈옥 등)에 취약한지 한눈에 보여주는 보안 성적표가 제공됩니다. 개발자는 이 취약한 답변들을 보고 시스템 프롬프트(System Prompt)에 "절대 ~한 답변은 하지 말 것"이라는 방어벽을 추가하면 됩니다.
💡 한 줄 요약
**redteam init**으로 내 모델과 원하는 취약점을 고르고, **redteam run**으로 모의 해킹을 보낸 뒤, **view**로 보안 뚫린 곳을 고치면 끝입니다!
